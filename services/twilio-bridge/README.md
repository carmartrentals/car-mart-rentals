# Car Mart Rentals — Twilio ↔ OpenAI Realtime Bridge

This is a tiny standalone Node.js WebSocket server. It exists because Vercel
serverless functions can't host long-lived WebSocket connections, and the
OpenAI Realtime API requires one.

```
Twilio Voice ─WSS─► [this bridge] ─WSS─► OpenAI Realtime
                          │
                          └─HTTPS─► carmartrentals.com (system prompt, tools, logging)
```

## What it does

1. Accepts an inbound Twilio Media Stream WebSocket (one per phone call)
2. Opens an OpenAI Realtime WebSocket and forwards G.711 µ-law audio in both directions
3. Calls back to the Vercel app for the system prompt, tool execution (send booking email, transfer, end), and final transcript logging
4. Tracks audio + text token usage per call so the app can compute exact per-call cost

## Deploy to Render (free tier, recommended)

1. Push this repo to GitHub (the bridge lives in `services/twilio-bridge/`).
2. Go to https://dashboard.render.com → **New** → **Blueprint**.
3. Point it at this repo. Render auto-detects `services/twilio-bridge/render.yaml`.
4. Set the `OPENAI_API_KEY` env var when prompted. `BRIDGE_SECRET` is auto-generated — copy it.
5. Once deployed, Render gives you a URL like `https://cmr-twilio-bridge.onrender.com`. Convert that to a WSS URL: `wss://cmr-twilio-bridge.onrender.com/twilio`.
6. In your Vercel app, set:
   - `BRIDGE_SECRET` = the same value Render generated
   - `TWILIO_BRIDGE_WSS_URL` = `wss://cmr-twilio-bridge.onrender.com/twilio`
7. In the CMR admin panel, go to **Settings → AI Receptionist** and switch **Voice Mode** to "Realtime (recommended)".

## Deploy to Fly.io (alternative)

```bash
fly launch --no-deploy
# accept defaults; sets up fly.toml
fly secrets set OPENAI_API_KEY=sk-... BRIDGE_SECRET=$(openssl rand -hex 32) APP_BASE_URL=https://www.carmartrentals.com
fly deploy
```

Note: Render's free tier sleeps after 15 min of inactivity. First call after a long quiet period will have ~30 sec of cold-start latency. If that's a problem, upgrade to Render Starter ($7/mo) or move to Fly.io which doesn't sleep.

## Local development

```bash
cd services/twilio-bridge
npm install
cp .env.example .env   # fill in OPENAI_API_KEY, APP_BASE_URL, BRIDGE_SECRET
npm run dev
```

Then expose your local port to the internet (so Twilio can reach it):
```bash
ngrok http 8080
```
Use the `wss://...ngrok.app/twilio` URL as your bridge WSS URL for testing.

## Env vars

| Variable | Required | Description |
|----------|----------|-------------|
| `OPENAI_API_KEY` | yes | Your OpenAI API key |
| `APP_BASE_URL` | yes | Public URL of the Vercel app, no trailing slash |
| `BRIDGE_SECRET` | yes | Shared secret with Vercel app — auto-generated by Render |
| `OPENAI_REALTIME_MODEL` | no | Defaults to `gpt-4o-mini-realtime-preview-2024-12-17` |
| `PORT` | no | Render/Fly set this automatically |
